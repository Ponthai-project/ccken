## 1. 課題提起：患者データの現実（山田氏パート）
- 「あなたの直近の血液検査データ、今すぐ出せますか？」という問いかけから開始。血液検査結果は紙でもらうことが多く、デジタルデータは手元になく自己管理できない実態。
- **患者側の課題**: 他院受診の際、自分のレントゲン画像であってもCD-Rに焼いて持参する必要があり、発行手数料も患者負担。家族のデータも一元管理できない。
- **病院側の課題**: 患者数×診療履歴の蓄積による管理コスト増、取り扱いミス時に病院が全責任を負う構造、患者に返したくても技術的・制度的な手段がない、という「患者でも管理できない、病院でも管理したくない」誰も得しない構造。
- **医療データの二次活用を阻む「同意」の壁**: 個別同意が必要だが同意取得が紙・対面のみでデジタル基盤がなく、スケールしない（数百〜数千件の同意収集が困難）。
- 経済産業省「医療機器産業ビジョン2024」を引用し、日本のSaMD（Software as a Medical Device）開発が世界に遅れている現状を指摘（モバイルアプリの臨床試験登録数: 日本23件 対 米国1,006件。AI技術を用いた医療機器: 日本75件 対 米国188件・中国204件）。

## 2. AIBTRUST社の医療プラットフォーム構想
大阪大学発ベンチャーAIBTRUST株式会社が「Trust 医療データの流通で日本の医療の発展を支える」をミッションに、AI・ブロックチェーン・Web3技術を基盤とした次世代医療インフラを構築。
- **医療スタッフ向けAI業務支援**: AI搭載型問診・カルテ作成支援、ダイナミックコンセントによる合法的なAIデータ活用、国際標準規格準拠AI問診によるデータ連携、RAGによる診療支援（過去の診療情報・ガイドラインから最適な医療方針をAIが提案）。
- **患者向けスマートフォンアプリ**: 複数病院の診察券・予約・検査結果の一元管理、家族の医療情報管理（親子機能）、ブロックチェーンによる改ざん防止、マイナンバー認証による本人確認。

山田氏の研究領域は医工学・情報ネットワーク・ロボティクスを基盤とし、日本のSaMD開発の遅れという課題に対し、ブロックチェーン・スマートコントラクト等のWeb3.0技術で解決を図るというビジョン。

## 3. SaMDのAIエージェント開発におけるアーキテクチャ上の最大リスク（PNG読み取り）
AIエージェント型開発を医療分野に適用する際、単なる技術的課題を超えた「トラスト（信用）」に関わる2大要件がアーキテクチャ上の大きな障壁になるとされる。

| リスク | 内容 |
|---|---|
| 同意管理のブラックボックス化 | 「いつ・誰が・何の目的でデータ利用に同意したか」が静的で、リアルタイムな追跡・制御が不可能な状態 |
| 実行証跡の改ざんリスク | AIエージェントが「どのデータに基づき、どう判断したか」という目的外利用防止の監査ログが、中央集権で改ざん可能な状態で保存されているリスク |

### AI駆動型開発を成立させる両輪：「主権のシフト」と「絶対的な証明」（PNG読み取り）

| 比較軸 | 従来型システム | AIエージェント・アーキテクチャ |
|---|---|---|
| 同意手法 | 静的・紙面ベース | ダイナミック・リアルタイム |
| データ主権 | 病院・医療機関 | 患者本人への移譲 |
| 実行証跡 | ブラックボックス | AI監査ログ・完全追跡 |
| トラスト基盤 | 中央集権型DB | 分散型台帳・ブロックチェーン |

### ダイナミックコンセントによる「リアルタイム・コントロール」（PNG読み取り）
データ提供の主権を患者本人へ移譲（主権のシフト）し、「どのデータを、誰に、いつまで貸すか」を患者自らが主体的に管理する基盤。デモ画面「Control Dial」では、誰に（AIエージェントA／専門病院B）、どのデータを（血液検査結果／レントゲン画像）、いつまで（24時間〜治療終了まで）を患者がダイヤル操作で制御するUIイメージが示された。個人情報保護法上の「第三者提供リスク」を根本から解消し、インシデントの開放につなげるという主張。

### ダイナミックコンセントとAI監査をつなぐ絶対的トラスト（PNG読み取り）
4ステップの技術フロー：
1. **同意ステータスの確認** — AIがダイナミックコンセントDBを参照し、患者のリアルタイムなアクセス許可を確認
2. **AIエージェントの実行** — 許可された範囲内でのみ、AIが医療データを処理（SaMDとしての推論）
3. **実行ログの生成** — 「どのデータを、何の目的で使用したか」を正確に記録したAI監査ログを生成
4. **トレーサビリティ管理** — ブロックチェーン技術を用いて、ログを改ざん不可能な状態で強固にロック

「目的外利用を防止するためには、AIの実行プロセス（監査ログ）とダイナミックコンセントの履歴を紐付け、改ざん不可能な基盤（ブロックチェーン）でトレーサビリティを管理することが必須要件となる」という結論。ダイナミックコンセントとブロックチェーンによるトラスト基盤は実証実験の段階を終えたとされる。

## 4. 医療分野におけるAIを「安全に使い続ける」ための管理体制（中野氏パート）
- 「原因を追跡できるか」「安全な管理を追跡できるか」「性能劣化などを追跡できるか」という3つの問いを、「記録・評価・説明できる体制」の構築によって解決するという方針。
- KDDIアイレットは、AIマネジメントシステムの国際規格「**ISO/IEC 42001**」を取得している。
- **AIを安全に使い続けるための継続的な管理サイクル（4要素）**:
  - **記録**: 利用履歴の記録／プロンプト・入出力／モデル・設定の変更／ログの保管・管理
  - **評価**: リスク評価／品質・性能の確認／継続的な監視／監査・レビュー対応
  - **説明・統制**: 説明責任の履行／規制・法令への対応／安全性の証明／ガバナンスの強化
  - **改善**: ガードレールの強化／運用プロセスの見直し

### AIMS（AI Management System）を「機能させる」ための記録と可視化基盤
- ルールが守られているかを可視化する仕組みが、AIMSの実効性を担保するという考え方のもと、「運用・実行」「監視・チェック」「改善・継続的向上」のサイクルを回す。
- テレメトリの収集・標準化・転送 → 蓄積・加工・分析基盤 → 可視化・スコアリングという流れで、AI Agent（Amazon Bedrock）や開発環境のコーディングエージェント（Amazon Bedrock）に関わるテレメトリを扱う構成が示された。
- **開発で使うAIも、AIMSの基準で管理**という方針のもと、5段階の管理サイクルが示されている:
  1. AIDD（仕様・テスト・整合性駆動）で開発を体系化
  2. 全てのエージェントのテレメトリを収集
  3. **指示・ハーネス・ツール**使用ログを可視化し改善
  4. IDPで開発者が迷わず安全なプロセスに乗れる
  5. AIMSの基準で作る過程も監査・管理下に

  > 注記: 本セッションでは「**指示・ハーネス・ツール使用ログを可視化し改善**」という記述があり、「ハーネス」の語が明示的に登場する（p27相当）。これはKDDIアイレット独自のAIMS運用管理フレームワーク内での用語であり、AI-DLC（AIM221/DVT452）側で使われる語ではない点に留意。

## 5. AWS Frontier Agentsによる運用保守の自動化
- 人手に頼る運用保守は障害検知・ログ分析への多大な人手と時間、属人化によるナレッジ喪失リスク、夜間休日対応体制維持の困難、対応遅延によるビジネスインパクト拡大という課題を抱える。
- **Frontier Agentsを利用したAI運用保守のアーキテクチャ**: 検知層（Amazon CloudWatch）→ DevOpsパイプライン層（AWS Lambda, DynamoDB, EventBridge等）→ GitHub連携層（AWS DevOps Agent Role, Claude Code, GitHub Actions）→ 運用・対応層、という多層構成（AWS Lambda, CloudWatch, DynamoDB, EventBridge, S3, WAF, SQS等のAWSサービスを活用）。
- **定量的効果**: 調査(411件)・起票(411件)・PR作成(326件)を対象に、人手対応の想定工数（約411h/68h/163h）に対しAIエージェント運用時（人間確認時間含む）は約154h/32h/52hに削減。人手想定 約643時間/月 に対しAIエージェント運用時 約238時間/月、月間約405時間（約51人日分）の工数削減を実現。

## 7. まとめ
医療という高度で専門的な領域でAIを活用していくには、安全に使い続けられる体制を構築することが重要という結論。
- **医療データの民主化**: ダイナミックコンセント×ブロックチェーン技術による医療データの民主化
- **AIMS準拠の管理体制**: ライフサイクル全体でAIの使い方を管理・記録・評価する枠組み
- **AIによる運用保守**: AIエージェントによる運用保守の自動化により、運用保守に関わる工数を約63%削減

締めのメッセージ: 「同意の壁を超え、医療データの民主化を実現し、医療の現場に、安心してAIを届けることで、SaMD開発が遅れる根因を取り除きます。」
