## 1. AIで加速する開発とそのリスク

- AIによって「作る」は速くなった（コード記述・調査・設計案整理・プロトタイプ・テスト・リファクタリング）。ただし「動き続ける」ものを作るには依然考慮すべき点が多い。
- **問い: アウトプットは2倍になったが、アウトカムは2倍になったか？**
- **AI時代のビルドトラップ**: アウトカムではなくアウトプット（機能の開発・リリース）で成功を計測しようとして行き詰まる状況。AIは「増幅器」（DORAレポート2025）→作るコストが下がることで、ユーザーが求めていないものを速く大量に作るリスクも高まる。
- 機能は作った瞬間から運用コスト（バグ修正・仕様変更・問い合わせ・QA・整合性維持・コンテキスト増大・ユーザーの認知負荷）が発生。機能が増えるほど複雑になり開発スピードは低下する。
- → AI時代において**「作るべきものを見極める力」がますます重要**。鍵はエンジニアがユーザー課題にDeep Diveし、プロダクトディスカバリーを主導・関与すること。

## 2. プロダクトディスカバリーとは

問題と解決策を探索し「何を作るか」を見極めるフェーズ（＝良いアイデアと悪いアイデアを迅速に判別する）。

| | Discovery | Delivery |
|---|---|---|
| 役割 | 何を作るかを見極める | 決めたものを作る |
| 重視 | 迅速な学習 | 安定性・スケーラビリティ |
| 目的 | 不確実性を下げる | ユーザーに届ける |
| 主導 | 主にPdM・デザイナー | 主にエンジニア |

- なぜ必要か: プロダクト開発失敗の根本原因は「ユーザーが求めていないものを時間をかけて作ること」。開発（ビルド）はアジャイルでも全体がウォーターフォールだとリスクが最後に来る。アイデアの少なくとも半分はうまくいかない。
- 目的（4つのリスクに対処し迅速に妥当性を立証）: 価値のリスク／ユーザビリティのリスク／実現可能性のリスク／事業実現性のリスク。
- 2フェーズ（ダブルダイヤモンド）: 正しい問題を見つける（ユーザーインタビュー・データ分析）＋正しい解決策を見つける（ブレスト・コンセプトテスト・プロトタイピング・A/Bテスト）。

## 3. エンジニアがディスカバリーで発揮する価値

- 従来ディスカバリーでは「作らず済むにこしたことはなかった」（コンシェルジュ実験・オズの魔法使い・モックアップ）が、AIで作るコストが劇的に下がり「作らず検証するコスト ≒ 作って検証するコスト」に。
- 「作るべきか/どう作るか」を判断できるのはエンジニア。「作る力」と「コンテキスト把握」があるからこそ適切な判断ができる。
- 従来PdM・デザイナー・リサーチャーが主導していたディスカバリーを、**テックリード/プロダクトエンジニアが主導・積極関与**することで価値を発揮（職種でなく課題の性質に応じて価値を発揮できる人が主導）。
- エンジニアが発揮する6つの価値: ①課題を深く理解し解決策を考えられる ②実現可能性を早期に見極められる ③作るかどうかの判断精度が上がる ④仮説検証の最適な落とし所がわかる ⑤AIのアウトプットが適切か判断できる ⑥分析結果から即座に次の仮説検証を回せる。

## 4. マイベストでの事例

- **事例① Web接客機能**: オフラインの接客体験をオンラインで。エンジニアが主導し、ユーザーインタビュー→元販売員ヒアリング→カテゴリを絞って検証→テーブルもAPIも作らずFrontendのみで簡易実装（**実工数1人日**）→A/Bテスト。リスクの高い部分から検証（約22%が使用、CVR5%向上）。
- **事例② favlist（新規事業）**: 会議室でどれだけ議論しても正解は出ない。ユーザーから学習する速度を最大化。立ち上げ期は事業責任者がPdMを担ったが、途中からエンジニアがPdMを兼務→コミュニケーションコストが減りコンテキストが集約され、何を作るかの判断がスピーディーに。
- **事例③ AI-DLC**: AWSが提唱、開発プロセス全体をAI前提で再構築しAIが実行を主導、人間が監督と意思決定。Inception（AIがビジネス意図を要件・ストーリーに変換、チーム全体で検証＝モブエラボレーション）／Construction（モブコンストラクション）／Operation。**AIのアウトプットが速いためゴッドPRDになりがち→エンジニアがスコープを抑制**し、検証のための最適なMVPに絞る。

## 5. AI時代に広がるエンジニアの価値と役割

**「作る力」から「何を作るべきかを見極め、学習速度を上げる力」へ**。技術力を土台に、ユーザー課題にDeep Diveして、品質・コスト・スピード・スコープのトレードオフを適切に判断し、アウトカムに繋げていく。

- AIで作る速度は上がったが、価値に辿り着く速度はそのままでは上がらない。
- エンジニアがディスカバリーを主導することで価値探索のスピードを加速できる。
